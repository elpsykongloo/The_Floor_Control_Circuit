# MiniCPM-o / Freeze-Omni 有效上下文应力测试结论

日期：2026-07-20

## 一、结论

本轮实测没有在 MiniCPM-o 4.5 或 Freeze-Omni 的官方位置上限附近发现 Moshi
式骤变。两模型都完整覆盖了项目拟用的 600 秒预算，期间未出现非有限值、KV
缓存意外缩短、隐藏状态范数尖峰、方向突跳或跨输入公共吸引态。

建议将“模型有效上下文”和“项目分析窗”分开登记：

| 模型与制式 | 官方位置上限 | 本轮安全采样位置 | 保守有效时长 | 项目建议 |
| --- | ---: | ---: | ---: | --- |
| MiniCPM-o，无主动滑窗 | 40960 | 40951 | **1279.28 秒（约 21.32 分钟）** | E1 仍取 **600 秒** |
| MiniCPM-o，仅倾听流量 | 40960 | 40951 | 3149.00 秒（约 52.48 分钟） | 只作描述性上界 |
| Freeze-Omni | 32768 | 32767 | **2618.24 秒（约 43.64 分钟）** | E1 仍取 **600 秒** |
| MiniCPM-o，`basic` 主动滑窗 | 高水位 8000，低水位 6000 | 每次淘汰后保留 5980 个动态位置 | **约 186.88–460.00 秒滚动历史** | 当前项目默认关闭 |

位置上限直接来自本机模型配置：

- `C:\artificial_intelligence\models\Full-Duplex\MiniCPM-o-4.5\config.json`：
  `max_position_embeddings=40960`；
- `C:\artificial_intelligence\models\Full-Duplex\Freeze-Omni\_assets\Qwen2-7B-Instruct\config.json`：
  `max_position_embeddings=32768`。

MiniCPM-o 每秒消耗 13–32 个主干位置。保守值按最密集的 32
位置/秒换算，并扣除 14 个固定提示位置：

```text
(40951 - 14) / 32 = 1279.28125 秒
```

Freeze-Omni 每 160 毫秒输入 2 个主干位置，即 12.5 位置/秒；扣除
39 个固定提示位置：

```text
(32767 - 39) / 12.5 = 2618.24 秒
```

因此，`文档/03_w3+w4计划.md` 的 D1 规则
`min(600 秒, 模型 context 规格)` 对这两个模型都应落为 **600 秒**。若制作
Moshi、MiniCPM-o、Freeze-Omni 三模型严格同窗的横向图，公共时间窗受 Moshi
限制，应另取 **240 秒**。

## 二、与 Moshi 的对照

Moshi 的已登记证据为：

- 官方 `C=3000` 步，对应 240 秒；
- 第 3000 行出现 80/80 会话一致的替代 sink 尖峰，与初始状态方向余弦约
  0.9985；
- 第 6000 行后深层跨会话公共方向占比由约 0.50 升至 0.85；
- T4 优势由 0–6000 步的 `+0.02401` 翻转为 6000 步后的 `-0.06006`，
  两段置信区间完全分离。

本轮沿用同一思路，在计划窗、`C`、`2C` 两侧加密采样，并同时检查隐藏状态、
跨输入公共方向、原生决策、KV 长度与动态 key。两模型在 `C` 处均未形成两类
证据联合触发的病理，在探索性 `2C` 处也没有发现相同形态。

## 三、MiniCPM-o 实测证据

### 3.1 官方上限

三条不同真实音频输入均运行到 40960 以上；联合分析有 195 个共同采样位置，
范围为 27–41081。位置 40960 前后：

- 四个抽样层的隐藏状态范数倍率为
  `1.0213 / 0.9986 / 1.0419 / 1.0003`；
- 最小边界前后方向余弦为 `0.9963`；
- 跨输入两两余弦增量均为负，未向单一公共方向突然收敛；
- 未中心化第一主方向占比约 `0.826–0.909`，该水平在边界前已经存在，边界后
  没有同步跃升；
- 全部记录为有限值，缓存缩短次数为 0，动态 key 余弦约为 1；
- 三条运行最终位置与缓存长度一致。

官方四段输入路径、单元内批处理路径和跨单元分组路径均完成对拍。跨单元分组
对逐单元路径的隐藏状态余弦为 `0.9999261`，logits 余弦为 `0.9999910`，
argmax 一致。

### 3.2 双倍上限探索

三条不同输入继续运行到 82044，联合覆盖 `2C=81920`。在 81920 前后：

- 四层范数倍率位于约 `0.976–1.011`；
- 最小方向余弦为 `0.9951`；
- 跨输入两两余弦最大增量约 `0.0222`；
- 没有非有限值、缓存缩短或多证据骤变。

该结果说明 MiniCPM-o 在本轮输入制式下越过 `2C` 仍保持工程稳定。正式规格
仍以 40960 为限；越界结果只作为反证 Moshi 型骤变的探索证据。

### 3.3 主动滑窗

`basic` 模式在逻辑位置 8009 首次触发预期淘汰：

```text
淘汰前：逻辑位置 7996，缓存长度 7996
淘汰后：逻辑位置 8009，缓存长度 5994，位置偏移 2015
```

扣除 14 个固定位置后，保留 5980 个动态位置，对应 186.88–460.00 秒历史。
边界附近最小隐藏方向余弦为 `0.9767`，四层范数倍率位于
`0.9595–1.0380`，没有联合病理。动态 key 变化符合主动淘汰预期。该模式允许
累计会话继续增长，同时较老 KV 已被移除，因此其“有效上下文”应表述为滚动
保留历史。

## 四、Freeze-Omni 实测证据

### 4.1 官方上限

三条不同真实音频输入均越过 32768；联合分析有 193 个共同采样位置，范围为
41–32831。位置 32768 前后：

- 四个抽样层的隐藏状态范数倍率为
  `1.0801 / 1.1541 / 1.0263 / 0.7903`；
- 最小边界前后方向余弦为 `0.9696`；
- 跨输入两两余弦最大增量约 `0.0207`；
- 未中心化第一主方向占比约 `0.529–0.572`；
- 决策熵由约 `0.4791` 变为 `0.4340`，跨输入决策一致率保持 `0.667`；
- 全部记录为有限值，缓存缩短次数为 0，动态 key 余弦约为 1。

官方首块输入和诊断直连路径实现了逐值对拍：隐藏状态最大绝对差为 0，状态概率
最大绝对差为 0，argmax 与缓存长度均一致。

### 4.2 双倍上限探索

受运行时和显存预算约束，`2C=65536` 只完成一条真实输入，最终位置为 65599。
该输入四层范数倍率位于 `0.933–1.050`，最小方向余弦为 `0.9500`，没有非有限
值或缓存缩短。单输入无法计算跨输入公共吸引态，因此这部分只构成数值稳定性
佐证，不能达到三输入官方边界测试的证据等级。

## 五、判定规则

分析器对每个候选边界、每个层同时计算：

1. 隐藏状态范数倍率与基于 MAD 的稳健跃迁；
2. 边界前后平均方向余弦；
3. 至少三条输入间的平均两两余弦与第一主方向占比；
4. 原生决策熵、跨输入一致率和可选任务 AUC；
5. KV 长度、位置偏移、动态 key 余弦与非有限值。

只有至少两类表征或性能证据联合触发，或出现非有限值，才确认上下文病理。
主动滑窗单列为结构淘汰。阈值、逐层中间量和判定结果均写入
`analysis.json`，便于独立复算。

## 六、证据边界

本轮使用真实音频建立输入嵌入库，并用三条独立音频检查跨输入行为；循环项没有
配套的冻结 floor 任务标签，因此任务 AUC 为缺失值。MiniCPM-o 在强制倾听制式
下的原生决策从早期就接近饱和，决策熵不能充当独立任务性能证据。

由此可以较强地回答缓存容量、数值稳定性和表征骤变问题；对“冻结 floor 探针
在多长时间后仍保持同等 AUC”的回答仍需用循环对齐标签补跑。运行器已支持
`--bank-labels`、顺序输入和逐逻辑步采样，后续无需重写统计代码。

当前最稳妥的研究决策为：

- MiniCPM-o 与 Freeze-Omni 的 E1 缓存窗口均采用 600 秒；
- 模型特定附录分别报告 1279.28 秒和 2618.24 秒的保守位置预算；
- MiniCPM-o 主分析关闭主动滑窗；若研究无限会话，另立 `basic` 滚动窗口条件；
- 不依据 `2C` 外推运行修改官方 context；
- 若要把这些结果写入冻结规格表，先按项目约定登记 `PREREG.md` 变更记录。
