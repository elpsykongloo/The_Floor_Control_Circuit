# MiniCPM-o 4.5 严格密集全双工上下文测量结论

日期：2026-07-20

## 结论

MiniCPM-o 4.5 在本实验中的上下文宽度需要分成三层报告：

1. **正式规格宽度**：模型配置为 `max_position_embeddings=40960`。默认系统提示占
   14 个位置，严格密集负载固定消耗 32 位置/秒，因此连续预算为
   `(40960 - 14) / 32 = 1279.5625` 秒；按完整的一秒单元计，正式规格内可完整容纳
   **1279 秒**。
2. **局部绝对位置外推**：三条不同音频、171 组三臂观测均未在 2600 秒内确认持续的
   多指标退化。实测干净下界为 **至少 2600 秒、约 83279 个位置**。该结论表示近期
   64 秒内容放到高绝对位置后，原生决策分布和生成行为仍大体稳定。
3. **完整密集历史的运行行为**：完整长流与同绝对位置的近期 64 秒后缀在
   **768 秒**开始出现持续分叉；最后一个未确认分叉的检查点是 **512 秒**。这表明
   长期累积内容已经显著改变当前发言决策。

所以，目前证据**不能保证 MiniCPM-o 在 2000 秒内保持充分的完整上下文能力且完全
没有能力退化**。模型可以运行到 2000 秒，局部高位置行为也通过了本协议，但 2000 秒
超出正式规格约 56%，完整长历史在此前已经发生稳定的分布与行为分叉。

工程上建议采用：

- 对需要正式规格保证的严格密集会话，截断上限取 **1279 个完整秒**；
- 对需要完整历史与近期窗口保持可比决策行为的保守运行，优先把 **512 秒**视为已经
  实测的安全检查点，并把 **768 秒**登记为本负载下的持续分叉起点；
- 2600 秒结果只作为局部 RoPE 位置外推证据，不能替代远端语义检索保证。

## 为什么使用三角对照

早期的长流—短后缀两臂比较会同时改变远端内容和绝对位置，容易把合法的历史影响
误判成位置失效。本轮在每个自然探针上运行三路：

1. **完整长流**：保留从起点到当前检查点的全部 KV；
2. **同位置近期后缀**：只重放最近 64 个完整密集单元，并动态平移 RoPE，使当前
   探针位置与完整长流逐项相同；
3. **低位近期后缀**：重放完全相同的 64 个单元，使用低位默认位置。

由此得到两个相互独立的问题：

- 同位置近期后缀 vs 低位近期后缀：测绝对位置压力；
- 完整长流 vs 同位置近期后缀：测远端历史累积效应。

零平移对拍共 3 组，隐藏状态最大绝对误差为 0，logits 最大绝对误差为 0，生成
token 全部逐项相同，证明动态位置控制没有在零平移条件下引入额外差异。

## 严格密集负载

每个一秒单元通过 MiniCPM-o 官方 `streaming_prefill` 与 `streaming_generate` 路径
产生：

```text
<unit>                 1 个位置
一秒音频嵌入          10 个位置
模型正文与特殊 token  19 个位置
<|chunk_eos|>          1 个位置
</unit>                1 个位置
合计                  32 个位置/秒
```

密集填充阶段只屏蔽会提前结束单元的 `listen`、`chunk_eos`、`chunk_tts_eos` 和
`turn_eos`；正文 token 仍按原始 logits 贪心生成。自然探针不做屏蔽。TTS 解码关闭，
因为它位于 LLM token 决策之后，不写回 Qwen KV。

检查点为：

```text
64, 256, 512, 768, 1024, 1200, 1260, 1280, 1300, 1400,
1600, 1800, 1950, 2000, 2050, 2200, 2400, 2560, 2600 秒
```

网格覆盖正式 40960 位置边界、2000 秒双侧窗口和 `2C=81920` 附近。每个检查点使用
3 个自然音频探针，读取第 8、17、26、35 层隐藏状态，并保存全词表 logits 与完整
贪心 token 序列。

### 绝对位置结果

位置臂随绝对位置增加出现了隐藏表征漂移；到 2600 秒，最差层余弦降至 0.8246，
全词表中心化 logits 余弦为 0.9386。原生决策分布和贪心行为保持稳定：

- 2600 秒的 JS 中位数为 0.000459，低于 0.05 阈值；
- 特殊决策总变差中位数为 0.001325，低于 0.10 阈值；
- 首 token 一致率为 1.000；
- 生成序列相似度中位数为 1.000；
- 三路有限值比例均为 1.000；
- 跨输入余弦增量与公共方向增量均未达到 0.20 阈值。

因此只有“表征”家族触发，分布、行为和数值家族没有形成相邻检查点的联合触发。
本轮没有观察到类似 Moshi 超上下文时多指标同时骤变的绝对位置悬崖。

### 完整历史结果

完整长历史比较在 512 秒只有表征家族触发，尚未达到确认条件。768 秒起，表征、
分布和行为三个家族同时触发，并在后续检查点持续：

- 768 秒：历史 logits 余弦 0.6515、JS 0.5022、特殊决策总变差 0.8798、首 token
  一致率 0.444；
- 2000 秒：历史 logits 余弦 0.4480、JS 0.6931、特殊决策总变差 0.99999、首 token
  一致率 0；
- 2600 秒：历史 logits 余弦 0.4550、JS 0.6931、特殊决策总变差 0.999997、首 token
  一致率 0。

这是一条运行行为边界。完整长流包含数百秒被强制维持高发言密度的模型输出，远端
内容改变当前决策可能具有合理语义。因此该结果支持“完整历史与近期窗口已显著分叉”，
尚不能单独证明远端信息已经不可检索。

2000 秒检查点的实际位置范围为 64033–64113，中位数为 64066，明确覆盖目标位置
`14 + 2000 × 32 = 64014`。

- 最差层隐藏余弦：0.8806；
- logits 余弦：0.9581；
- JS：0.00000259；
- 特殊决策总变差：0.00000956；
- 首 token 一致率：1.000；
- 生成序列相似度中位数：1.000；
- 最大跨输入余弦增量：0.0993；
- 最大公共方向增量：0.1005。

完整长流与同位置近期后缀之间：

- 最差层隐藏余弦：0.1574；

- JS：0.6931；

这组证据支持“2000 秒时局部高位置决策仍稳定”，同时明确反对“2000 秒完整上下文
能力无退化保证”这一强结论。

## 判定规则

绝对位置退化按四类证据登记：

1. 表征：隐藏余弦或 logits 余弦同时跨过绝对阈值和早期基线下降阈值；
2. 分布：JS 与特殊决策总变差同时跨过阈值；
3. 行为：首 token 一致率或完整序列相似度跨过阈值；
4. 数值：任一路出现非有限值。

确认退化要求至少两个位置指标家族触发，并在两个相邻检查点持续；任意非有限值可
立即确认。完整历史分叉另行允许历史结构指标参与，且不进入绝对位置退化裁决。

完整阈值和实现见
`src/floor_circuit/minicpm_dense_context.py` 与
`文档/05_MiniCPM-o上下文最终结论.md`。

## 解释边界

1. 近期高位后缀检验的是局部 RoPE 位置外推，不能证明模型能从数千秒前准确检索
   任意语义事实。
2. 完整长流保留了远端 KV，但密集负载中的长期强制发言本身会改变对话状态；历史
   分叉同时包含正常内容效应和潜在上下文能力变化。
3. 生成音频只来自三个独立输入构建的有限嵌入库，覆盖不了所有语言、说话人和噪声
   条件。
4. 本实验测量原生决策分布、隐藏状态和生成 token；冻结的 floor 标签 AUC需要在
   E1 协议中另行复算。
5. 2600 秒超限运行成功不能扩张模型正式规格。
