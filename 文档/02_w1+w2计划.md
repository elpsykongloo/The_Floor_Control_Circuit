```
src/floor_circuit/        # 共享库，跑在本仓库 uv 环境（Python 3.12）

configs/                  # 冻结参数 YAML：events.yaml / grids.yaml / stimuli.yaml
                          #   / paths.windows.yaml / splits/（划分文件，生成即冻结）
```

3. **划分文件生成即冻结**：`configs/splits/*.json`（含生成种子）提交入库，之后只读。
4. 所有脚本带 `if __name__ == "__main__":` 守卫；音频 IO 统一走环境内 ffmpeg（00 §11 Windows 注记）。

## 3. 工作包

### WP1 事件管线 + G0 校准

**实现**（参数照 00 §2 冻结，写入 `configs/events.yaml`）：

- 双通道 VAD：Silero-VAD v5，threshold 0.5，min_speech 120 ms，min_silence 180 ms。
- IPU：同通道连续语音，内部间隙 < 180 ms 合并。
- Turn：EN 启发式（向 CANDOR Backbiter 对齐，间隙 < 1.0 s 合并、backchannel 不切断）；ZH 以 SmoothConv 专家标签为准。backchannel 词表按 00 §2.1。

**输出 schema**：每会话一份 `events.parquet`（通道、事件类型、t_start/t_end、置信度、参数指纹）+ 帧级标签表（按目标模型时钟重采样）。

**单元测试**：合成波形用例（静音—语音—重叠—短间隙的构造序列，事件真值已知）；SmoothConv 抽 20 段与专家标注人工核对差异清单。

**G0 校准**：

### WP5 激活缓存系统

- 格式：zarr（`.gitignore` 已屏蔽）、fp16、按层分组、时间维分片（约 4096 步/片）；每 run 一份 `manifest.json`（模型、模式 R1/R2、层列表、hidden 维度、时钟 Hz、种子、温度、源音频路径与哈希、代码版本）。
- 预算核算：单层 ≈ 100 KB/s（4096 维 fp16 @ 12.5 Hz）。**MVE = 4 层 × 200 段 × ≤10 min ≈ 48 GB**，NVMe 无压力；全量 E1 的 2 TB 上限与隔层预案（00 §4.4、§12.2）本冲刺不触发。
- 位置：`D:\...\activations\moshi\mve_r1\<session_id>\`。
- 前向耗时预估：≈33 h 音频的 teacher-forced 前向（无生成采样循环），bf16 单卡按接近实时估 ≈30–40 GPU·h，D7 起跑、跨夜完成（估算值，D7 实测后修正）。

### WP6 刺激库 S1–S5 全量 + 质检（W1 启动 V6，W2 完成；规格照 00 §5 冻结）

| 库 | 规格 | 来源与工具 |
| --- | --- | --- |
| S1 语义完整性最小对 | 300 对/语言（完整 vs 不完整前缀）；同音色合成；时长配平 ±5%、响度 ±0.5 LU；各配 F0 拉平版；ZH 另裁 SmoothConv 真实 complete/incomplete 段（合成+真实双轨） | Qwen3-TTS（V6 定音色）；parselmouth（F0 拉平）；pyloudnorm |
| S2 打断内容最小对 | 5 类各 ≥ 200 trials：(a) 指令性打断 (b) 能量/时长匹配 babble (c) (a) 倒放 (d) backchannel (e) 跨语言版；onset 在宿主语句 20–80% 均匀采样 | (a)(d)(e) Qwen3-TTS；(b) MUSAN（`D:\dataset\audio\MUSAN`）或音素乱序重合成；(c) 波形倒放 |
| S3 停顿场景库 | N=200/语言，turn 中段 300–1200 ms 静默 | EN 自 CANDOR PAUSE 事件；ZH 自 incomplete 段末（依赖 WP1/WP2 产出的事件表） |

- 统一交付：24 kHz、固定种子；manifest（id、类别、语言、文本、时长、LUFS、来源、变换参数、哈希）→ `D:\...\stimuli\<S*>\<lang>\` + `manifest.parquet`。
- 质检门：全量自动核（采样率、时长差 ±5%、响度差 ±0.5 LU、削波率）+ **5% 人工抽听**；S1 需过文本模板句法核对（完整/不完整前缀二人工抽检）。不合格项重合成，记录轮数。
- 按各模型输入格式的预编码缓存**推迟到 E2 前**，本冲刺不做（V2 所需的单段 Mimi 试验除外）。
- 交付：`reports/stimuli_qc.md`（各库通过率、重合成轮数、抽听结论）。

### WP7 MVE（Moshi，R1）与 G1

**配置**（照 00 §6-E0 第 5 步冻结）：

- 模式：R1 观察者（teacher-forced 复放）；模型 moshiko-pytorch-bf16；缓存温度 Transformer 层 **{4, 12, 20, 28}** 残差流 @ 12.5 Hz。
- 数据：CANDOR **200 段**、每段前 10 min = 160 段（探针训练划分内）+ 40 段（验证划分内），会话级与全局划分一致；因果评估侧 25% 不触碰。双通道对称各作一次 agent（标签按 00 §2.2 生成；bootstrap 的 cluster 单位始终 = 会话）。
- 目标与标签：
  - **T1 发言起始预测**：{LISTEN, GAP} 态每个时钟步，δ = 240 ms（= 3 步，满足 H1 的 ≥3 时钟步前瞻要求），标签 = ONSET_agent ∈ (240, 320 ms]。
  - **T4 话轮终点投射**：对方 IPU 末步二分类 {complete, incomplete}（EN 启发式 TURNEND，00 §2.2）。
- 探针协议：线性 logistic（L2 正则，C ∈ {0.01, 0.1, 1} 在验证段上选）；特征 = 残差流向量（fp16→fp32），按层 z-score（统计量只在训练段拟合）；3 种子；负类下采样 5:1；报 AUC + AUPRC + balanced accuracy；会话级 cluster bootstrap 1,000 次 → 95% CI（00 §8）。
- 基线族（三基线 + sanity，全部同划分、同评估段）：
  1. **声学特征 GRU**：80 ms 帧 {RMS, F0, 谱通量, ZCR}，2 s 上下文（25 步）单层 GRU(128) + logistic 头（torch，本仓库 uv 环境）；
  2. **Mimi 潜表征线性探针**：量化前连续潜表征 @12.5 Hz（若不可得则码本嵌入求和），同探针协议；
  3. **hazard-rate 生存模型**：离散时间 hazard logistic（协变量：自上次 ONSET/OFFSET/说话人切换的时间、当前态时长、当前态类型）；
  4. shuffled labels（sanity，期望 AUC ≈ 0.5）。
- **G1 统计量**：每目标 = 最优层探针 AUC − max(基线 1–3 AUC)；T1 与 T4 双行并报；**裁决取两者较大者**，并要求该目标的会话级 bootstrap 95% CI 下界 > 0。

**G1 分支（照 00 §7）**：

| 结果 | 行动 |

| ≥ +0.05 | 全量 E1（W3 起五模型展开） |
| +0.02 ~ +0.05 | PersonaPlex 与 MiniCPM-o 各补一次 MVE（脚本在 WP4 已就绪，预算 +3 天）；任一 ≥ +0.05 → 全量 E1，主受试换过线模型 |
| 全部 < +0.02 | 分支 N1：止损改组（00 §12.3），冻结代码与报告后重估选题 |

**交付**：`reports/mve_报告.md`（4 层 × 2 目标 × {探针, 3 基线, shuffled} 的 AUC/AUPRC 表 + CI + G1 裁决与分支行动）。
