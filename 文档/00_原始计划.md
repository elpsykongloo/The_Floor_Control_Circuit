# The Floor-Control Circuit: Locating and Steering Turn-Taking Decisions in Full-Duplex Speech Models

状态：**冻结**（2026-07-17）。本版取代 v1.0。目标：ICLR 2027（摘要 ≈ 2026-09 下旬）。
环境：Windows 本地，模型与数据均已配置完毕。

**冻结配置**

- 英文受试模型：Moshi 7B（主）、PersonaPlex-7B-v1（同架构对照）
- 中文受试模型：MiniCPM-o 4.5（端到端原生）、Freeze-Omni（模块化 L1 对照）
- 附加对照：dGSLM（无文本流，仅探针级）
- 数据：CANDOR（EN）、SmoothConv（ZH 金标）、DuplexConv（ZH 规模）、dualturn-switchboard-turn-taking（EN 帧级标签）

## 1. 核心主张与假设体系

### 1.1 一句话

在开放权重全双工语音模型内部，定位"话语权（floor）决策"的表征，因果验证之，将其做成免微调的推理时行为旋钮，并判定该决策的信息基础（语义 vs 声学）；同时回答该表征是否是跨架构（L1/L2）、跨语言的收敛解。

### 1.2 假设与预注册判据

- **H1（可解码性与前瞻性）**：中后层存在对未来交互事件的线性可解码信息。
  判据：最优层线性探针 AUC − max(声学特征基线, 编解码/编码器表征基线, hazard-rate 基线) ≥ **0.05**，会话级 cluster-bootstrap 95% CI 下界 > 0；在该模型自身决策时钟的 ≥ 3 个时钟步前瞻仍成立。
- **H2（因果性）**：对 H1 子空间做 patching/方向注入可翻转持有/让位决策。
  判据：翻转率 ≥ **50%** 且高出范数匹配随机方向 ≥ **30 pt**；剂量单调（Spearman ρ > 0.8）；内容方向对照无效（双重分离）。
- **H2b（整合器动力学）**：floor 投影的时间演化服从对用户语音活动证据的漏积分；拟合时间常数可预测让位延迟分布形状（KS 检验不拒绝）。
- **H2c（通路中介）**：归因 patching 预筛 + 真实 patching 验证的 top-k 用户流监测头，中介 barge-in→让位效应的比例可量化（中介效应比 ± CI），并给出发言门控的直接 logit 归因读出通路。
- **H3（可控性与优越性）**：沿 floor 方向的推理时 steering 存在工作区间。
  判据：存在 α 使停顿处理 TOR 相对基线下降 ≥ **30%**、打断响应 TOR 不降、GPT 内容分下降 ≤ **5%**、UTMOS 下降 ≤ 0.15；SDT 分解上 d′ 保持率不劣于 PAD/静音 logit 偏置（若等价，如实报告并转述为"无需打开黑箱"的实用结论）。
- **H4（信息基础）**：floor 决策对语义完整性敏感、超越能量线索。
  判据：语义最小对引发显著行为与表征差分（置换检验 p < 0.01），能量匹配对照不引发；"懂该语言"的模型差分显著大于"不懂该语言"的模型（2×2 交互项显著）。**H4 全线不成立 → 触发 N3 反例论文路线（§12.3）**。
- **H5（收敛性，扩展版）**：
  - (a) 同架构（Moshi↔PersonaPlex）方向可移植且效力与行为质量同向；
  - (b) 跨架构（Moshi 系 ↔ MiniCPM-o ↔ Freeze-Omni 骨干）floor 子空间几何相似（CKA/Procrustes 显著高于随机匹配），即使不可直接移植；
  - (c) 双语模型（MiniCPM-o、Freeze-Omni）内部， EN/ZH 共享同一 floor 子空间（跨语言探针迁移 AUC 降幅 ≤ 0.05）。

### 1.3 主张阶梯（claim ladder）

- **Tier 1（底线）**：子空间级因果 + 控制（H1–H4）。
- **Tier 2（正文机制通路）**：head 级定位与"监测 → 整合 → 门控"通路草图，含 H2b/H2c 的量化（§6-E2 Tier-2 工作包）。
- **Tier 3（条件触发的电路图主张）**：仅当稀疏度门 G3b 通过（top-8 头中介 ≥ 70%，头身份跨 3 种子 × 2 语料 Jaccard ≥ 0.6）才写入正文并补 faithfulness 电池；否则标注为后续工作。
- **默认不主张**：最小完备电路作为标题级贡献（时间展开决策、双通道读出、时间性自修复使完备性在本任务上未良定义，见 §12.4-Q9）；闭环产品级效果。

## 2. 事件本体与操作化（参数冻结）

统一在各模型**自身决策时钟**上分析；跨模型比较一律换算为毫秒。

### 2.1 基础检测

- 双通道 VAD：Silero-VAD v5，threshold 0.5，min_speech 120 ms，min_silence 180 ms。
- IPU：同通道连续语音，内部间隙 < 180 ms 合并。
- Turn（EN 启发式，向 CANDOR Backbiter 对齐）：同说话人相邻 IPU 间隙 < 1.0 s 且期间对方无非-backchannel IPU 则合并；对方 ≤ 1.0 s 且命中 backchannel 判据的 IPU 不切断 turn。ZH 侧以 SmoothConv 专家标签为准。
- Backchannel 词表：EN {yeah, uh-huh, mm, mhm, right, okay, oh, wow, sure, i see}；ZH {嗯, 嗯嗯, 对, 对对, 是, 是的, 好, 哦, 啊, 行}。ZH 优先用 turn=backchannel 金标覆盖。

### 2.2 事件定义（agent 通道 X，对方通道 Y）

| 事件      | 定义                                                         |
| --------- | ------------------------------------------------------------ |
| ONSET_X   | VA_X 0→1，此前 X 静默 ≥ 400 ms                               |
| OFFSET_X  | VA_X 1→0，此后 X 静默 ≥ 400 ms                               |
| YIELD_X   | Y 在重叠中 onset 后，X 于 ≤ 1.0 s 内 OFFSET                  |
| HOLD_X    | 同上情形但 X 继续 ≥ 1.5 s                                    |
| BC_Y      | Y 的 IPU ≤ 1.0 s、命中 backchannel 判据、X 的 VA 未中断      |
| GRAB_Y    | Y 在 X 说话中 onset 且 X 随后 YIELD                          |
| PAUSE_Y   | Y turn 内静默 ∈ [300 ms, 2.0 s]（ZH 金标准：incomplete 段末静默） |
| TURNEND_Y | ZH：complete 段末；EN：IPU 末且 Y ≥ 1.5 s 无续说，或 X ≤ 1.0 s 内 onset |

### 2.3 探针目标

- **T1 发言起始预测**：{LISTEN, GAP} 态每个时钟步，标签 = ONSET_agent ∈ (δ, δ+一个时钟步]。
- **T2 让位预测**：Y-onset-in-overlap 后各步，标签 = agent 于 400 ms 内 OFFSET。
- **T3 来话分类**：Y onset + 2 个时钟步处，三分类 {backchannel, 抢话, 其他}。
- **T4 话轮终点投射**：Y 的 IPU 末步，二分类 {complete, incomplete}。
- **T5 状态解码（sanity）**：五分类 {SPEAK, LISTEN, OVERLAP→yield, OVERLAP→hold, GAP}。

### 2.4 前瞻量网格（按模型时钟）

| 模型                | 决策时钟                                                     | δ 网格                                                       |
| ------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| Moshi / PersonaPlex | 80 ms（Mimi 12.5 Hz）                                        | {0, 80, 160, 240, 400, 800 ms}                               |
| dGSLM               | 其离散单元帧率（以官方发布为准）                             | 换算为 ≈{0, 100, 200, 400, 800 ms} 最近步                    |
| Freeze-Omni         | chunk 末帧（VAD 触发的 chunk 流；chunk 长以官方默认配置为准） | {0, 1, 2, 3 chunk}                                           |
| MiniCPM-o 4.5       | LLM 以 1 Hz 决策是否发言；骨干 token 位置更细                | 决策级 {0, 1, 2 s}；骨干位置级另设 {0, 250, 500, 1000 ms} 双轨报告 |

### 2.5 运行制式

- **R1 观察者**（teacher-forced 复放）：双通道语料喂入，事件 = 人类事件。测"模型表征了什么"。
- **R2 代理**（受控自由运行）：user 通道逐步注入语料/刺激音频，agent 侧自由生成（固定温度与种子），事件 = 模型自身决策。**H2/H2b/H2c/H3 全部只在 R2 做**。Moshi 系用官方推理管线改造（PersonaPlex offline eval 脚本为骨架）；Freeze-Omni/MiniCPM-o 各用其官方本地 demo/server 改造（FDB v1.5 即以本地部署 Freeze-Omni 官方 demo server 的方式评测，先例成立）。
- **R3 闭环抽查**：各模型官方实时链路 + Qwen3-TTS 用户模拟器，每配置 10 段，仅验证真流式一致性。

行为读出（R2，双通道并报）：token 级（Moshi 系 = 文本流 P(PAD) 与静音音频 token 概率；MiniCPM-o = 1 Hz 发言决策 + 交错文本/语音 token；Freeze-Omni = 状态头 logits，官方答案）+ 信号级（生成音频 80 ms 帧 RMS + VAD）。

模型特定设计要点：

- **Freeze-Omni 三重角色**：(a) 正对照——其状态头就读末层，故我们的探针在末层必然能恢复状态信息，管线若做不到即有 bug；(b) 科学问题——floor 信息在**冻结的文本 LLM** 中沿层如何浮现（它只能由语音编码器适配路径注入，"哪一层开始可读"刻画注入-加工深度）；(c) E3 的 L1 官方旋钮——扫描状态头判决阈值/logit 偏置，得到模块化范式的控制 Pareto，与原生 steering 正面对比。
- **MiniCPM-o 4.5 两重角色**：(a) 中文原生受试（探针/因果/操控全做，δ 网格按 1 Hz 决策时钟缩减）；(b) 跨架构收敛性检验——与 Moshi 系比较 floor 子空间几何（CKA、Procrustes 对齐后余弦），不做直接移植（架构不同）。W1 需验证其音频-only 流式模式与官方 demo 链路的可挂钩性（V1，见 §14）。
- **PersonaPlex 亮点实验**：同架构方向移植（PersonaPlex 方向 → Moshi 各层对应注入），检验能否修复 Moshi 已量化的停顿抢话缺陷。

## 5. 刺激工程（Qwen3-TTS，零人工标注）

- **S1 语义完整性最小对**：文本模板生成 300 对/语言（句法完整前缀 vs 不完整前缀），Qwen3-TTS 同一音色合成（EN、ZH 各一固定音色），时长配平 ±5%、响度配平 ±0.5 LU（pyloudnorm）；各配 F0 拉平版（praat-parselmouth）。ZH 另有 SmoothConv complete/incomplete 真实段裁剪（合成受控 + 真实金标双轨互证）。
- **S2 打断内容最小对**：agent 说话中固定时刻注入：(a) 指令性打断（EN "wait, stop, that's wrong" / ZH "等等，停一下，不对"），(b) 能量/时长匹配 babble（MUSAN 或音素乱序重合成），(c) (a) 的时间倒放，(d) backchannel（"yeah"/"嗯嗯"），(e) 跨语言版（ZH 打断喂 EN 模型、EN 打断喂 ZH 分析）。每类 ≥ 200 trials，onset 在 agent 语句 20–80% 处均匀采样。
- **S3 停顿场景库**：turn 中段截出 300–1200 ms 静默（EN 自 CANDOR PAUSE 事件；ZH 自 incomplete 段末），N=200/语言。
- **S4 应答场景库**：完整问句 + 静默（EN 启发式；ZH complete 金标），N=200/语言。S3+S4 构成 SDT 的两类。
- **S5 韵律/信道操纵**：对 S2(a) 做音高 ±4 半音、语速 ±20%、加性噪声 SNR ∈ {20, 10, 5 dB}。

全部 24 kHz 交付、按各模型输入格式预编码缓存、种子固定；刺激生成脚本随论文发布。

## 6. 实验设计（E0–E5，逐个定稿）

### E0 基础设施与 MVE

**目的**：管线可信 + 最低成本判断 H1 是否值得全量。
**步骤**：

1. 事件管线（§2 参数）实现，在 DualTurn-SWB 官方标签上校准。
2. SmoothConv/DuplexConv JSON 解析器 + turn 标签映射；CANDOR Backbiter 话轮导入。
3. 四个 R2 运行器：Moshi 系（PersonaPlex offline eval 骨架）、MiniCPM-o（官方 demo 链路改造）、Freeze-Omni（官方 server 改造）。
4. 逐层激活缓存 hook（每模型 1 段音频做维度/对齐 sanity）。
5. **MVE**：CANDOR 200 段 → Moshi R1 缓存层 {4, 12, 20, 28} → T1(δ=240 ms) 与 T4 线性探针 vs 三基线（声学特征 GRU、Mimi 潜表征探针、hazard-rate 生存模型）。
   **输出**：管线校准报告；MVE AUC 表。
   **判据**：见 G0/G1（§7）。

### E1 可解码性与几何（H1）

**输入**：R1+R2 缓存（Moshi 全层；PersonaPlex 全层；MiniCPM-o 骨干全层；Freeze-Omni 骨干全层；dGSLM agent 塔全层）。

**程序**：

1. 全层 × δ 网格 × T1–T5 线性探针；每目标加 2 层 MLP 对照（线性度差距）。R1 与 R2 各一套（R2 标签 = 模型自身事件，是后续因果的靶）。
2. 基线族（每模型配齐）：声学特征（RMS、F0、谱通量、ZCR + 2 s 上下文 GRU）；"编码器/编解码表征"探针（Moshi 系 = Mimi 潜表征；MiniCPM-o = Whisper 编码输出；Freeze-Omni = 语音编码器输出）——若骨干内部不超过编码器输入端，则"LLM 未加工交互信息"；hazard-rate 生存模型；shuffled labels。
3. 几何：类条件均值 PCA；子空间秩扫描（rank-k 投影后性能曲线 → 有效维度）；事件对齐平均轨迹（ONSET/YIELD 前 2 s 的决策变量爬升，Fig 3）。
4. 泛化矩阵：跨会话（CANDOR 内）；跨语料（CANDOR↔DualTurn-SWB）；跨语言输入（EN 探针 → ZH 音频，逐模型）；SmoothConv→DuplexConv（标注噪声稳健性）。
5. 外部上界并列：多语种 VAP（EN/ZH 现成模型）与 DualTurn 报告值。
6. Freeze-Omni 专项：状态头信息沿层浮现曲线（末层为正对照锚点；报告"最早可读层"）。
   **对照**：上文基线族全套。
   **输出图表**：Fig 2（层 × δ AUC 热图 × 5 模型，基线等值线）、Fig 3（轨迹与几何）、Table 2（探针主结果矩阵）。
   **判据**：G2（§7）。

### E2 因果定位（H2/H2b/H2c）

**受试**：Moshi（全量）→ PersonaPlex（复现）→ MiniCPM-o（ZH 缩减网格：仅 ℓ* 附近 5 层 × 1 个窗口档）。dGSLM 不做。Freeze-Omni 骨干做一组"注入位点"patching（把 floor 信息在早层擦除，看状态头是否失明——检验骨干内表征相对状态头的上游性）。
**程序（Moshi 全量版）**：

1. **Patching**：捐体 = R2 中 YIELD 前 400 ms 窗口激活；受体 = 匹配上下文的持续说话运行。层 ℓ × 窗口 w ∈ {1, 3, 10 步} × 粒度 {全残差, floor 子空间, 正交补}。读出 = 未来 1 s 内 P(PAD) 变化 + 生成音频 VA。反向（持有→让位情形）对称做。
2. **方向消融**：S2a 刺激到达时投影清零 floor 分量 → 预测"不让位"；量化让位率变化。
3. **剂量**：注入强度扫描 → 剂量-反应曲线。
4. **Tier-2 通路**：EAP 归因 patching 预筛全部头与 MLP（每 trial ≈ 2 前向 + 1 反向）→ top-32 真实 patching（每头 ≥ 200 trials，head-resampling 对照）→ 中介效应比 ± CI → 路径 patching（用户流位置 → 监测头 → floor 子空间）→ PAD/静音 token 直接 logit 归因（经文本头与 Depth Transformer 的门控读出）。
5. **H2b 动力学**：floor 投影拟合漏积分器（对用户 VA 证据），时间常数预测让位延迟分布，KS 检验。
6. **ITI 式头级干预**：验证过的 top-k 头输出上施加方向干预，对照残差流位点。
   **对照（预注册）**：范数匹配随机方向 ×20；话题/情感探针方向（双重分离）；错层（早层）与错时（事件前 2 s）；内容特异性（后续 5 s 文本流困惑度、语义相似度）。
   **试次**：每条件 ≥ 300 trials（翻转率 CI 半宽 ≤ 6%）。
   **方法学限定（写入论文）**：自回归流式生成存在时间性自修复（t 步消融的决策可在 t+1 步经他路重算），必要性结论一律窗口化表述。
   **输出图表**：Fig 4（层×窗口翻转热图、剂量曲线、双重分离柱状）、Fig 4b（中介效应比 + 通路示意 + 漏积分拟合）。
   **判据**：G3、G3b（§7）。

### E3 推理时操控（H3）＋范式级对比

**受试**：Moshi（全量）、PersonaPlex（复现 + 提示旋钮）、MiniCPM-o（ZH，α 三档）、Freeze-Omni（阈值旋钮基线）。
**程序**：

1. 方向构造：差分均值（YIELD 前窗 − HOLD 窗；TAKE 前窗 − 继续倾听窗），基于 R2 自身事件；备选 LDA 白化。
2. 施加：(i) 持续 h←h+α·σ_ℓ·v̂；(ii) 事件触发（检测到用户 onset 后 1 s 窗）。α ∈ {±0.5, ±1, ±2, ±4}。
3. 行为评测：FDB v1 四维 + FDB v1.5 四类重叠场景协议 + 语料复放统计（gap/overlap 分布对 CANDOR/SmoothConv 人类分布的 JSD）；内容与音质：文本流困惑度、GPT-judge、UTMOS。
4. **SDT 分解**：S3/S4 场景集，decision = 1.2 s 内 onset；沿 α 画 ROC，分解 d′ 与 criterion c。
5. **五旋钮对比（定稿版）**：① floor-steering（本文）；② PAD/静音 logit 偏置；③ 提示控制（PersonaPlex 角色提示改写"耐心倾听者/急切参与者"；MiniCPM-o 系统提示同理）；④ LoRA 微调上界锚点（moshi-finetune rank 128；若本地显存不支持则标注为云端可选项，不阻塞主线）；⑤ **Freeze-Omni 状态头阈值扫描**（L1 模块化范式的官方旋钮）。比较维度：d′ 保持率、可达 c 范围、内容损失、所需资源、跨场景稳定性——产出"原生表示级控制 vs 输出级偏置 vs 提示级 vs 模块阈值级"的范式对比表（Table 4，论文卖点之一）。
6. R3 闭环抽查：每配置 10 段（Qwen3-TTS 用户模拟器）。
   **输出图表**：Fig 5（α 系列 FDB 雷达、交互-内容 Pareto、SDT d′/c 曲线、五旋钮对比）。

### E4 信息基础分离（H4，2×2 定稿设计）

**设计**：{EN 刺激, ZH 刺激} × {懂该语言的模型, 不懂的模型}，同一音频跨模型对照。

- EN 语义对 → Moshi/PersonaPlex（懂）与——不存在"不懂 EN"的受试，用 (c) 倒放与 (b) babble 充当 EN 的"剥离"对照；
- ZH 语义对 → MiniCPM-o/Freeze-Omni（懂）vs Moshi/PersonaPlex（词汇剥离）：**同一刺激、四模型并测**，交互项 = 词汇通路贡献。
  **程序**：

1. S1/S2/S5 刺激下行为差分（让位率、响应延迟）× 表征差分（floor 投影时间曲线）。
2. 分层语义分歧曲线：语义对 vs 能量对的表征差异随层数演化（语义信息在第几层进入 floor 通路）。
3. 中介分析：语义完整性 →（floor 投影）→ 发言决策，直接 vs 中介效应。
4. Freeze-Omni 状态头 = 模块化系统对同刺激的"官方判决"，与其骨干内部表征对照。
5. dGSLM-lite：S2 行为差分（无文本流条件下语义打断是否仍有差分——文本支架必要性）。
6. 合成-真实一致性：S1 合成对 vs SmoothConv 真实段（ZH）。
   **输出图表**：Fig 6（最小对行为差分；逐层分歧曲线；2×2 交互图；心理测量曲线）。
   **判据**：G4（§7）。

### E5 收敛性与综合（H5）

1. Moshi vs PersonaPlex：全管线复现比较（可解码性、线性度、因果效力、方向余弦/主角度）；**方向移植**：PersonaPlex 方向 → Moshi，能否修复停顿抢话（Fig 7 主图）。
2. 跨架构几何：Moshi 系 ↔ MiniCPM-o ↔ Freeze-Omni 骨干的 floor 子空间 CKA / Procrustes 对齐相似度，对照随机子空间分布。
3. 双语模型内跨语言：MiniCPM-o 与 Freeze-Omni 上 EN 探针 ↔ ZH 探针互测（迁移 AUC 降幅 ≤ 0.05 判据）；EN/ZH 方向余弦。

## 8. 统计规范

**摘要 A 版（≈150 词）**：

> Full-duplex speech models must continuously decide when to speak, yield, and tolerate backchannels, yet these decisions have only been evaluated behaviorally. We give the first causal, representation-level account of floor control across five open models spanning token-synchronized (Moshi, PersonaPlex, dGSLM) and hidden-state-predictor (Freeze-Omni) architectures, plus an end-to-end omni model (MiniCPM-o 4.5), in English and Chinese. Layer-wise probes reveal a low-dimensional floor state anticipating the model's own onsets and yields beyond acoustic and encoder baselines. Activation patching shows it causally gates speaking, doubly dissociating from content directions; its dynamics fit a leaky integrator whose time constant predicts yield latencies. Adding a scaled floor direction at inference yields a continuous interruptibility dial that repairs pause-grabbing with minimal content cost, compared head-to-head against logit bias, prompting, and modular threshold control. Same-stimulus cross-lingual tests quantify the semantic versus acoustic basis of the decision, and transplanting the direction from PersonaPlex into Moshi repairs its pause handling.

**摘要 B 版（H4 不成立时）**：

> …Energy-matched minimal pairs, time-reversed speech, and same-stimulus cross-lingual tests show the decision is driven by acoustic evidence: models that understand the language behave and represent indistinguishably from models that do not. Internally decodable floor information does not exceed what the audio encoder already provides. Full-duplex "semantic turn-taking," as currently trained, is behaviorally and representationally equivalent to a learned voice-activity detector — with direct implications for the native-versus-modular architecture debate.

| 风险                                                 | 概率           | 应对                                                         |
| ---------------------------------------------------- | -------------- | ------------------------------------------------------------ |
| G1 失败（可解码性不足）                              | 低-中          | 备胎 MVE（PersonaPlex/MiniCPM-o）；全败走 N1 早止损          |
| 表征可读不因果（G3 失败）                            | 中             | N2 伴随现象论文：E1+E4+外部对比重构；干预位点先换（attention/MLP 输出）再判死 |
| H4 失败（纯能量驱动）                                | 中             | N3 反例论文（摘要 B），E1–E3 全部复用，影响力不降            |
| steering 工作区间过窄                                | 中             | 诚实 Pareto；贡献重心移向 E2/E4；logit-bias 等效本身可发表   |
| Moshi 事件类稀缺（停顿抢话 TOR≈1 致 YIELD 稀少）     | 中             | S2 刺激人为诱发让位事件；PersonaPlex 作事件富集源            |
| MiniCPM-o 音频-only 流式挂钩受阻                     | 低-中          | V1 第一周验证；受阻则其角色降级为 E1/E4（探针+行为），ZH 因果侧由 Freeze-Omni 骨干 patching 顶替 |

| dGSLM 生成退化                                       | 高             | 设计上已限定为探针级；不影响任何 Gate                        |

### 12.2 工程风险

激活缓存 IO 瓶颈（→ 隔层缓存 + fp16 + 分片写 + 存到机械硬盘）；五模型环境冲突（→ 每模型独立 env + 统一数据接口层）；长音频 KV cache 显存（→ 本地有双卡，可以跨卡加载KV）。

### 12.3 分支论文预案

- **N1**："全双工模型内部交互信息不超过其音频编码器输入端"——需 E4 行为证据与 VAP/DualTurn 对比支撑；判定为弱产出，触发即重估选题。
- **N2**："可解码而不因果：全双工模型话轮表征是伴随现象"——E1 全量 + 双分离失败图 + 外部预测器对照，机制类负结果。
- **N3**："Glorified VAD"——最强的负结果分支：标题/摘要 B 版，E1–E3 全部保留（steering 重述为能量判据的表示级旋钮），直接裁决原生 vs 模块化之争。

### 12.4 审稿人 FAQ（预写反驳，9 条）

1. "probing 不是机制" → 核心证据在 E2/E3 因果与控制；探针只做定位；每张主图带负对照。
2. "可能只是 VAD" → H4 是一等公民：编码器基线 + 能量匹配 + 同刺激跨模型词汇剥离三重防线；且失败即 N3。
3. "模型选择偏倚" → 五模型横跨 L1/L2 与两语言；Moshi 系为当前唯一可全流程干预的开放原生家族，如实声明。
4. "steering 是旧技术" → 新在现象类别（交互策略）、因果链条完整性、SDT 行为刻画、五旋钮范式对比、跨模型移植。
5. "Moshi 行为本来就差" → 失败（停顿抢话）与成功（打断响应及时）并存 = 分析燃料；steering 目标恰是修复其已量化缺陷；PersonaPlex 提供好行为锚点。
6. "事件定义任意" → 参数冻结并在 DualTurn 官方帧级标签上校准；ZH 有专家金标。
7. "合成刺激分布外" → S1 双轨（Qwen3-TTS 受控 + SmoothConv 真实金标）互证。
8. "伦理" → 可打断性旋钮可被滥用为霸道 agent；发布默认安全区间与用途声明；数据依各自许可使用（SmoothConv CC BY-NC）。
9. "为什么不给最小完备电路图" → 粒度做到 head 级（Tier-2 中介量化 + ITI 式干预）；回避的只是完备/最小主张：决策多步展开、双通道读出使完备性未良定义；时间性自修复使必要性检验偏差不可控；faithfulness 指标脆弱与消融自修复已有文献；G3b 保留证据充分时当场升级的通道，且数据已全部采集。
