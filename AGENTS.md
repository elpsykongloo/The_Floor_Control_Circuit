# AGENTS.md — 智能体工作约定

任何在本仓库工作的 AI 智能体，请先完整阅读 `CLAUDE.md`（持久记忆与约定的权威来源，本文件与其冲突时以 `CLAUDE.md` 为准）。科学定义以 `文档/00_原始计划.md` 为权威，本机路径以 `文档/01_本地路径.md` 为权威，当前冲刺执行细节见 `文档/02_w1+w2计划.md`。

## 核心规则

1. **提交 = commit + push**：用户说"提交"或"推送"任意一个，都要既 `git commit` 又 `git push -u origin <当前分支>`（push 失败按 2/4/8/16 秒退避重试至多 4 次）。
2. **大文件不入 Git**：激活缓存、音频、权重、实验中间产物一律写 `D:\data_storage\The_Floor_Control_Circuit`；仓库只收代码、配置、文档、小报告、小图。
3. **中文写作**：文档、注释、commit message 用中文（专有名词与代码标识符保留英文）。
4. **冻结参数与判据不得擅改**：变更须先登记进 `PREREG.md` 变更记录，再改配置并重跑受影响的校准。
5. **环境分工**：模型与数据只在用户 Windows 本机可用（`C:\`、`D:\`）；云端会话无法访问，只做文档、代码、计划、审查，不得虚报"已跑通"。
6. **Python 环境**：本仓库统一用 uv（`uv run`，Python 3.12）；五个受试模型各用自己的 `.venv`（路径见 文档/01 §3）；不新建全局环境，不跨环境 import。
7. **不建 PR**，除非用户明确要求。

- 默认分支 `main`，工作分支 `claude/*`；commit message 中文、动词开头，涉及工作包/Gate 注明编号。
